# Описание информационной системы

## Назначение
Система автоматизирует учет индивидуальных достижений учащихся, модерацию заявок, формирование рейтинга и аналитики.

## Роли
- **Администратор**: управление пользователями, аудит, системные настройки.
- **Куратор**: проверка достижений и заявок на регистрацию, работа с секциями/новостями.
- **Ученик**: подача достижений, просмотр статусов, участие в секциях.

## Основные модули
- Авторизация и сессии
- Регистрация учеников
- Достижения и модерация
- Рейтинг и аналитика
- Календарь событий
- Новости и объявления
- Секции и заявки в секции
- Уведомления
- Аудит действий

## Данные
Сервер использует PostgreSQL, миграции в `server/migrations`.
Ключевые таблицы:
- `users`
- `refresh_sessions`
- `audit_logs`
- `registrations`
- `achievements`

## API
Контракт в `server/contracts/openapi.yaml`.
Базовый префикс: `/api/v1`.

## Развертывание
Рекомендуемый путь: Docker Compose (см. README).
