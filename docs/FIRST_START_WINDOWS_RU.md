# Запуск проекта с нуля на Windows

Документ написан для человека без технического опыта.

Если вы никогда раньше не запускали frontend, backend и базу данных, это нормально. В этом проекте их не нужно поднимать по отдельности. Всё запускается одной командой через Docker.

---

## Что вы получите после запуска

После успешного запуска у вас будут работать сразу 3 части проекта:

1. `web` — сайт, который открывается в браузере.
2. `api` — backend-сервер, который обрабатывает данные.
3. `db` — база данных PostgreSQL.

Отдельно запускать frontend и backend не нужно.

---

## Что нужно заранее

Перед началом проверьте, что у вас есть:

1. Компьютер с Windows 10 или Windows 11.
2. Доступ в интернет.
3. Доступ к репозиторию `IS_Diplom`.
4. Хотя бы 15-20 ГБ свободного места на диске.
5. Права на установку программ на компьютер.

---

## Шаг 1. Установите Docker Desktop

Docker Desktop нужен для того, чтобы проект сам поднял сайт, backend и базу данных.

Что сделать:

1. Откройте браузер.
2. Перейдите на сайт Docker Desktop:
   [Docker Desktop](https://www.docker.com/products/docker-desktop/)
3. Скачайте версию для Windows.
4. Запустите установщик.
5. Везде оставляйте стандартные настройки.
6. Если установщик предложит включить `WSL 2` или попросит перезагрузку — соглашайтесь.
7. После установки откройте `Docker Desktop`.
8. Дождитесь, пока программа полностью загрузится.

Как понять, что всё готово:
- в окне Docker Desktop не должно быть красных ошибок;
- обычно отображается статус вроде `Engine running` или `Docker Desktop is running`.

Важно:
- пока Docker Desktop не запущен, проект не стартует;
- если Docker только что установили, иногда нужно один раз перезагрузить компьютер.

---

## Шаг 2. Скачайте проект

Самый простой способ для новичка — не через Git, а через ZIP-архив.

Что сделать:

1. Откройте страницу репозитория `IS_Diplom` в GitHub.
2. Нажмите зелёную кнопку `Code`.
3. Нажмите `Download ZIP`.
4. Дождитесь завершения скачивания.
5. Найдите ZIP-файл в папке `Загрузки`.
6. Нажмите по нему правой кнопкой мыши.
7. Выберите `Извлечь всё...`.
8. Укажите папку, куда хотите распаковать проект.

Рекомендация:
- лучше распаковать проект в короткий путь, например `C:\Projects\IS_Diplom`
- так меньше шанс получить проблемы из-за длинных путей

После распаковки зайдите в папку проекта и убедитесь, что внутри есть:

- `docker-compose.yml`
- `README.md`
- папка `server`
- папка `src`

Если этих файлов нет, значит вы открыли не ту папку. Нужно открыть папку, в которой лежит именно проект.

---

## Шаг 3. Откройте PowerShell прямо в папке проекта

Теперь нужно открыть окно команд именно в папке проекта.

Что сделать:

1. Откройте папку проекта в Проводнике Windows.
2. Кликните мышкой по адресной строке сверху.
3. Полностью сотрите текущий путь.
4. Напишите слово `powershell`.
5. Нажмите `Enter`.

После этого откроется синее окно PowerShell.

Это очень важный шаг: команды нужно вводить именно в этом окне и именно в папке проекта.

---

## Шаг 4. Проверьте, что Docker работает

В открывшемся PowerShell по очереди вставьте команды:

```powershell
docker --version
docker compose version
```

Что должно произойти:
- появятся строки с версиями Docker;
- не должно быть ошибки вроде `docker is not recognized`.

Если появилась ошибка:

1. Закройте PowerShell.
2. Откройте Docker Desktop.
3. Подождите 1-2 минуты.
4. Снова откройте PowerShell в папке проекта.
5. Повторите команды.

---

## Шаг 5. Запустите проект

В этом же PowerShell выполните главную команду:

```powershell
docker compose up --build
```

После этого начнётся запуск проекта.

Что в этот момент делает компьютер:

1. Скачивает нужные Docker-образы.
2. Собирает frontend.
3. Поднимает PostgreSQL.
4. Запускает backend.
5. Backend автоматически применяет миграции базы данных.
6. Backend автоматически загружает стартовые данные.
7. Запускает веб-часть проекта.

Это нормально, если первый запуск идёт долго.

Обычно первый старт занимает от 5 до 15 минут, иногда чуть больше, если интернет медленный.

Важно:
- не закрывайте это окно PowerShell, пока работает проект;
- не пугайтесь, если по экрану бегут логи — это нормально;
- если всё работает, окно должно оставаться открытым без аварийного завершения.

---

## Шаг 6. Откройте проект в браузере

Когда запуск завершится, откройте браузер и перейдите по адресу:

- сайт: [http://localhost](http://localhost)

Дополнительно можно проверить backend:

- backend health-check: [http://localhost:8080/health](http://localhost:8080/health)

Если backend запущен правильно, по адресу `/health` вы увидите ответ вида:

```json
{"ok":true}
```

---

## Шаг 7. Войдите в систему

Для первого входа можно использовать тестовые аккаунты.

Основные аккаунты:

- `admin / admin123`
- `curator / curator123`
- `student / student123`

Дополнительные аккаунты, которые уже добавлены в проект:

- кураторы: `curator01`, `curator02`, `curator03`
  пароль у всех: `curator123`
- ученики: `student01` ... `student24`
  пароль у всех: `student123`

Если сайт открылся, но вы видите старые данные или не видите новых пользователей:

1. Откройте сайт в режиме инкогнито.
2. Если не помогло — очистите данные сайта для `localhost` в браузере.
3. После этого обновите страницу.

Это нужно потому, что часть данных интерфейс сохраняет в браузере.

---

## Шаг 8. Как остановить проект

Если проект больше не нужен прямо сейчас, его можно остановить.

Что сделать:

1. Вернитесь в окно PowerShell, где работает проект.
2. Нажмите `Ctrl + C`.
3. Дождитесь остановки.
4. Затем выполните команду:

```powershell
docker compose down
```

Что делает эта команда:
- корректно останавливает контейнеры;
- не удаляет базу данных.

---

## Шаг 9. Как запустить проект в следующий раз

Когда захотите снова открыть проект, повторите только короткую последовательность:

1. Откройте Docker Desktop.
2. Дождитесь, пока он полностью загрузится.
3. Откройте PowerShell в папке проекта.
4. Выполните:

```powershell
docker compose up
```

Если вы меняли код проекта, лучше запускать так:

```powershell
docker compose up --build
```

---

## Шаг 10. Как полностью сбросить проект и начать заново

Если что-то сильно сломалось и вы хотите полностью начать с чистого состояния, выполните:

```powershell
docker compose down -v
docker compose up --build
```

Важно:
- ключ `-v` удаляет данные базы данных;
- после этого база будет создана заново;
- все локальные данные PostgreSQL будут потеряны.

Используйте этот вариант только если обычный запуск уже не помогает.

## Шаг 11. Что делать, если не запускается

### Ситуация 1. Команда `docker compose up --build` не работает

Проверьте:

1. Открыт ли Docker Desktop.
2. Дождались ли вы полного запуска Docker Desktop.
3. Находитесь ли вы в папке проекта.
4. Есть ли в текущей папке файл `docker-compose.yml`.

---

### Ситуация 2. Ошибка `docker is not recognized`

Это значит, что Windows не видит Docker.

Что делать:

1. Убедитесь, что Docker Desktop установлен.
2. Убедитесь, что Docker Desktop открыт.
3. Перезагрузите компьютер.
4. После перезагрузки снова откройте Docker Desktop.
5. Потом снова откройте PowerShell и повторите команду.

---

### Ситуация 3. Ошибка про занятый порт

Обычно это выглядит как сообщение о том, что порт уже используется.

В проекте используются такие порты:

- `80` — сайт
- `8080` — backend
- `5432` — PostgreSQL

Если порт занят, значит на компьютере уже работает другая программа, которая использует этот порт.

Что делать:

1. Закрыть лишние программы, которые могут использовать эти порты.
2. Если у вас уже установлен PostgreSQL, он может занимать `5432`.
3. После этого повторить запуск.

---

### Ситуация 4. Сайт открылся, но работает странно

Что проверить:

1. Откройте PowerShell в папке проекта.
2. Выполните:

```powershell
docker compose ps
```

Вы должны увидеть контейнеры `db`, `api` и `web`.

Потом можно посмотреть логи:

```powershell
docker compose logs api
docker compose logs web
docker compose logs db
```

Если не понимаете текст ошибок, сохраните эти логи и покажите человеку, который помогает с проектом.

---

### Ситуация 5. Всё очень долго запускается

Для первого старта это нормально.

Причины:

- Docker скачивает образы из интернета;
- frontend собирается впервые;
- база данных создаётся с нуля.

Если прошло меньше 15 минут, чаще всего нужно просто подождать.

---

## Шаг 12. Как понять, что frontend и backend действительно поднялись

Если проект стартовал правильно, происходит следующее:

1. В браузере открывается [http://localhost](http://localhost).
2. Backend отвечает по адресу [http://localhost:8080/health](http://localhost:8080/health).
3. В `docker compose ps` видны контейнеры `web`, `api`, `db`.
4. Можно войти под `admin / admin123`.

Именно это и означает, что проект поднят полностью.

---

## Самая короткая памятка

Если вам нужен совсем короткий вариант, то он такой:

1. Установить Docker Desktop.
2. Скачать проект ZIP-архивом.
3. Распаковать проект.
4. Открыть PowerShell в папке проекта.
5. Выполнить:

```powershell
docker compose up --build
```

6. Открыть [http://localhost](http://localhost).
7. Для остановки выполнить:

```powershell
docker compose down
```

---

## Главное, что нужно запомнить

Для этого проекта человеку без опыта не нужно отдельно запускать frontend, backend и базу данных.

Нужно сделать только 3 вещи:

1. Установить и запустить Docker Desktop.
2. Открыть PowerShell в папке проекта.
3. Выполнить `docker compose up --build`.

На этом проект должен подняться полностью.
