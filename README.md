# Банк одарённых детей — полноценная информационная система

Репозиторий содержит:
- **Frontend (React + Vite)**
- **Backend API (Node.js + Express)**
- **База данных PostgreSQL**
- **Миграции, seed, API-контракт (OpenAPI), CI**

Это позволяет скачать проект и развернуть его локально или на сервере как единую full-stack систему.

---

## Что здесь вообще происходит

Если вы «абсолютный ноль» в Docker, то самая простая идея такая:

- **frontend** — это сайт, который открывается в браузере;
- **backend** — это сервер, который принимает запросы от сайта и работает с данными;
- **database** — это PostgreSQL, где данные хранятся постоянно;
- **Docker** — это способ запускать всё это в готовых изолированных контейнерах, чтобы не ставить вручную кучу зависимостей.

Иными словами, вместо длинной ручной настройки Node.js, PostgreSQL, переменных окружения и отдельных процессов, вы запускаете одну команду, а Docker сам поднимает:
1. базу данных,
2. API-сервер,
3. веб-приложение.

---

## Архитектура проекта

- `src/` — клиентское приложение (кабинеты: админ/куратор/ученик)
- `server/src/` — сервер API
- `server/migrations/` — SQL-миграции БД
- `server/contracts/openapi.yaml` — API-контракт
- `docker-compose.yml` — единый запуск `web + api + db`
- `scripts/predeploy.sh` — скрипт, который проверяет проект и запускает контейнеры

## Самый простой способ запуска — через Docker

### Что нужно установить

#### Windows / macOS
Установите **Docker Desktop**.

#### Linux
Установите:
- Docker Engine,
- Docker Compose plugin.

После установки перезапустите компьютер или терминал, если система этого просит.

Проверьте, что всё установилось:

```bash
docker --version
docker compose version
```

Если обе команды отработали без ошибки, значит Docker готов к работе.

## Пошаговый запуск для новичка

### 1) Скачать проект
Если проект уже лежит у вас на компьютере, переходите к следующему шагу.

Если нет, откройте терминал и выполните:

```bash
git clone <URL_ВАШЕГО_РЕПО>
cd Site
```

---

### 2) Убедиться, что Docker запущен

На **Windows/macOS** обычно нужно просто открыть приложение **Docker Desktop** и дождаться, пока оно покажет статус вроде *Docker is running*.

На **Linux** можно проверить так:

```bash
docker info
```

Если команда не падает с ошибкой, значит демон Docker работает.

---

### 3) Собрать и запустить проект

В корне проекта выполните:

```bash
docker compose up --build
```

Что делает эта команда:
- `docker compose` — читает файл `docker-compose.yml`;
- `up` — запускает все сервисы проекта;
- `--build` — сначала заново собирает образы, чтобы запуск был из актуального кода.

При первом запуске это может занять несколько минут, потому что Docker:
- скачает базовые образы,
- установит зависимости,
- соберёт frontend,
- поднимет базу данных,
- запустит backend,
- запустит nginx/web-контейнер.

---

### 4) Что вы увидите после запуска

После успешного старта будут доступны:
- **Frontend:** `http://localhost`
- **API:** `http://localhost:8080`
- **PostgreSQL:** `localhost:5432`

Входные данные пользователей создаются администратором системы.

При запуске API автоматически:
1. ждёт, пока будет готова база данных,
2. применяет миграции,
3. загружает начальные тестовые данные.

---

### 5) Как остановить проект

Если запускали в текущем окне терминала, нажмите:

```bash
Ctrl + C
```

Если хотите корректно остановить контейнеры отдельной командой:

```bash
docker compose down
```

Если хотите **полностью удалить данные базы** и начать с чистого состояния:

```bash
docker compose down -v
```

Важно:
- `docker compose down` — остановит контейнеры, но сохранит том с БД;
- `docker compose down -v` — удалит ещё и том, то есть база будет создана заново при следующем запуске.

## Запуск «одной командой» через predeploy-скрипт

Если вы хотите не просто поднять контейнеры, а ещё и автоматически проверить проект перед запуском, используйте:

```bash
chmod +x scripts/predeploy.sh
./scripts/predeploy.sh
```

Скрипт автоматически:
1. проверит наличие Docker и Docker Compose,
2. проверит наличие `package-lock.json`,
3. выполнит `npm run check:contracts`,
4. выполнит `npm run check:store`,
5. выполнит `npm run check:types`,
6. выполнит `npm run build`,
7. проверит `docker compose config`,
8. запустит `docker compose up -d --build`.

Это хороший вариант, если вы хотите убедиться, что проект собирается «как положено».

## Что такое Docker Compose в этом проекте

Файл `docker-compose.yml` описывает три сервиса:

### `db`
Контейнер с PostgreSQL.
Он хранит данные приложения.

### `api`
Контейнер с backend на Node.js/Express.
Он подключается к базе и отдаёт API.

### `web`
Контейнер с frontend.
Он собирает и отдаёт клиентскую часть приложения.

Docker Compose поднимает их вместе и связывает между собой автоматически.
Поэтому backend внутри Docker обращается к базе не как к `localhost`, а как к сервису `db`.

## Что делать, если проект не стартует

### 1) Проверить список контейнеров

```bash
docker compose ps
```

Вы увидите, какие сервисы запущены и какие порты заняты.

### 2) Посмотреть логи

Все логи сразу:

```bash
docker compose logs
```

Логи только backend:

```bash
docker compose logs api
```

Логи только frontend:

```bash
docker compose logs web
```

Логи только базы:

```bash
docker compose logs db
```

### 3) Пересобрать проект с нуля

Иногда помогает такой сценарий:

```bash
docker compose down -v
docker compose up --build
```

Это удалит старые контейнеры и старую базу, а затем всё соберёт заново.

## Типичные ошибки и что они означают

### Ошибка: порт уже занят
Например, если заняты `80`, `8080` или `5432`.

Что делать:
- остановить другую программу, которая использует этот порт;
- либо поменять порты в `docker-compose.yml`.

### Ошибка: Docker не найден
Значит Docker ещё не установлен или не запущен.

Что делать:
- установить Docker Desktop / Docker Engine;
- перезапустить терминал;
- снова проверить `docker --version`.

### Ошибка: база не поднимается
Что делать:
- проверить `docker compose logs db`;
- убедиться, что порт `5432` не занят другой PostgreSQL;
- выполнить `docker compose down -v`, потом `docker compose up --build`.

### Ошибка: сайт открывается, но API не отвечает
Что делать:
- проверить `docker compose logs api`;
- убедиться, что контейнер `api` запущен;
- убедиться, что backend дождался готовности БД.

## Нужно ли что-то настраивать вручную?

Для запуска через Docker — **почти ничего**.

В текущей конфигурации в `docker-compose.yml` уже прописаны:
- порт backend,
- доступ к БД,
- имя базы,
- логин и пароль PostgreSQL,
- JWT secrets для локального запуска.

То есть для локальной разработки можно просто выполнить:

```bash
docker compose up --build
```

И этого достаточно.

## Если хотите запуск без Docker

Это сложнее для новичка, потому что нужно вручную установить и запустить:
- Node.js,
- npm,
- PostgreSQL,
- переменные окружения,
- frontend и backend в разных терминалах.

### Backend

```bash
cd server
cp .env.example .env
npm install
npm run migrate
npm run seed
npm run dev
```

### Frontend

```bash
cd ..
npm install
npm run dev
```

Такой режим полезен разработчикам, но для первого запуска лучше использовать Docker.
